CPU 스케줄링 알고리즘

CPU 스케줄러

프로그램을 실행하기위한 프로세스들을 관리해주는 존재

사용자가 요청한여러가지 작업들을 프로세스와 PCB를 생성하고 이 정보들로 어떤 작업을 언제 어떤 순서로 실행할지를 정해준다.

이러한 작업을 수행하기위해 CPU가 어떻게 이용당하는지(선점, 비선점), 어떤 상황이 나타나고 어떻게 해결할지를 정할 여러가지 알고리즘이 있고 이를 정리합니다.

CPU 스케줄링이 결정되는 시점

  1. 실행(Running) 상태의 프로세스의 상태가 바뀌는 경우

    • 실행(Running) => 대기(Waiting) => ex. 입출력 인터럽트(I/O Interrupt)가 발생했을 때, 비선점 방식에서
    • 실행(Running) => 준비 완료(Ready) => ex. 타이머 인터럽트(Timer Interrupt)가 발생했을 때, 선점될 때
    • 실행(Running) => 종료(Terminated)

  2. 레디 큐(Ready Queue)에 변화가 있는 경우

    • 대기(Waiting) => 준비 완료(Ready)

CPU 스케줄링 알고리즘

FCFS(First Come First Served)

  • 특징
    • 비선점형 스케줄링
    • 먼저 온 프로세스를 먼저 서비스 해주는 방식
  • 문제점
    • convoy effect 👀

SJF(Shortest - Job - First)

  • 특징
    • 비선점형 스케줄링
    • 다른 프로세스가 먼저 도착했어도 CPU burst time 이 짧은 프로세스에게 선 할당
  • 문제점
    • starvation 👀

SRTF(Shortest Remaining Time First)

  • 특징
    • 선점형 (Preemptive) 스케줄링
    • 새로운 프로세스가 도착할 때마다 새로운 스케줄링이 이루어진다.
  • 문제점
    • starvation 👀
    • 새로운 프로세스가 도달할 때마다 스케줄링을 다시해 CPU burst time(CPU 사용시간)을 측정할 수가 없다.?

Priority Scheduling

  • 특징
    • 우선순위가 가장 높은 프로세스에게 CPU 를 할당하는 스케줄링이다. 우선순위란 정수로 표현하게 되고 작은 숫자가 우선순위가 높다.
    • 선점형 스케줄링(Preemptive) 방식 더 높은 우선순위의 프로세스가 도착하면 실행중인 프로세스를 멈추고 CPU 를 선점한다. 우선순위가 같으면 FCFS로 진행한다.
    • 비선점형 스케줄링(Non-Preemptive) 방식 더 높은 우선순위의 프로세스가 도착하면 Ready Queue 의 Head 에 넣는다.
    • 우선순위를 정하는 방법
    • 내부적 우선순위 결정 제한 시간, 기억장소 요청량, 사용 파일 수, 평균 프로세서 버스트에 대한 평균 입출력 버스트의 비율
    • 외부적 우선순위 결정 프로세스의 중요성, 사용료를 많이 낸 사용자, 작업을 지원하는 부서, 정책적 요인
  • 문제점
    • starvation 👀
    • 무기한 봉쇄(Indefinite blocking) 👀
  • 해결책
    • aging 👀

    • Round Robin

    • 특징
      • 현대적인 CPU 스케줄링 (멀티태스킹을 많이 해서?)
      • 각 프로세스는 동일한 크기의 할당 시간(time quantum)을 갖게 된다.
      • 할당 시간이 지나면 프로세스는 선점당하고 ready queue 의 제일 뒤에 가서 다시 줄을 선다.
      • RR은 CPU 사용시간이 랜덤한 프로세스들이 섞여있을 경우에 효율적
      • RR이 가능한 이유는 프로세스의 context 를 save 할 수 있기 때문이다.(PCB 덕)
    • 장점
      • Response time이 빨라진다. n 개의 프로세스가 ready queue 에 있고 할당시간이 q(time quantum)인 경우 각 프로세스는 q 단위로 CPU 시간의 1/n 을 얻는다. 즉, 어떤 프로세스도 (n-1)q time unit 이상 기다리지 않는다.
      • 프로세스가 기다리는 시간이 CPU 를 사용할 만큼 증가한다. 공정한 스케줄링이라고 할 수 있다.
    • 주의

      time quantum의 길이를 신경써야 함.

      • 길때 : FCFS랑 다를게 없음
      • 짧을 때 : context switch도 비용이 있기 때문에 자주할수록 비용이 추가됨(overhead)

    개념 정리

    선점

    시분할 시스템에서 타임 슬라이스가 소진되었거나, 인터럽트나 시스템 호출 종료 시에 더 높은 우선 순위 프로세스가 발생 되었음을 알았을 때, 현 실행 프로세스로부터 강제로 CPU를 회수하는 것을 말한다.

    비선점

    일단 CPU 를 잡으면 CPU burst 가 완료될 때까지 CPU 를 반환하지 않는다. 할당되었던 CPU 가 반환될 때만 스케줄링이 이루어진다.

    startvation

    CPU burst time이 긴 프로세스는 영원히 CPU를 할당받지 못할 수 있다.

    convoy effect

    소요시간이 긴 프로세스가 먼저 도달하여 cpu를 혼자 점유하여 효율이 낮아지는걸 말함.

    무기한 봉쇄(Indefinite blocking)

    실행 준비는 되어있으나 CPU 를 사용못하는 프로세스를 CPU 가 무기한 대기하는 상태

    aging

    아무리 우선순위가 낮은 프로세스라도 오래 기다리면 우선순위를 높여주자.
