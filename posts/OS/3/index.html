프로세스와 쓰레드 차이

프로세스 (Process)

운영체제로부터 자원을 할당받은 작업의 단위

운영체제가 정적으로 저장만되어있는 프로그램들을 실행시키기위해 프로세스라는 작업단위로 프로세서를 운영한다.

프로세스는 해당 작업을 위해 운영체제로부터 준비, 대기, 실행 과같은 작업에대한 상태를 부여받고 실행상태가 되었을 때 자원을 할당받아 프로세스를 진행한다.

이때 이러한 상태나 프로세스 식별 등 프로세스에대한 정보를 PCB(Process Control Block)가 가지고 스케줄링에 사용된다.

프로세스가 할당받은 자원을 아래와 같은 형식으로 할당해준다.

  • 스택 : 프로세스로부터 스택형식으로 할당된 메모리 영역을 받고, 쓰레드별로 할당되어 쓰레드의 작동 관련 지역변수, 매개변수, 리턴값, 돌아올 주소 등등이 저장되고 함수 호출시 기록하고 종료되면 제거한다.

  • 스택형식 이외의 메모리에서 아래 3가지 형식으로 영역 공유

    • 힙 : 개발자가 필요할 때마다 사용하는 영역으로 런타임에 결정된다. c언어에서 동적 할당과 같이 사용하며 사용후 해제를 해야 메모리 누수가 일어나지 않는다.
    • 데이터 : 코드에서 선언한 전역변수 또는 static 변수 등등이 저장된 공간, Read-Write
    • 코드 : 프로세스가 실행할 코드와 매크로 상수가 기계어의 형태로 저장된 공간, 컴파일 타임에 결정, Read-Only

쓰레드

과거에는 프로세스하나가 실행 시작 ~ 끝까지 진행 했지만, 요즘 프로그램들은 복잡해지고 기능이 많아지며 프로세스 하나만을 사용해서 프로그램을 실행하기는 벅차다.

프로세스를 여러개 생성하기엔 운영체제 자체적으로 프로세스마다 자신에게 할당된 메모리만 접근이 가능하도록 제약이 있어 사용할 수 없고, 이를 해결하기위해 쓰레드가 생겼다.

쓰레드는 프로세스를 대신하기위해 생긴 개념이라 동작과 특성은 비슷하지만 프로세스 내부에서 쓰레드끼리 자원을 공유하면서 작업을 진행할 수 있다.

차이점을 정리하면

  • 프로세스는 실행중 죽어도 다른 프로세스에 영향이 없다. 프로세스는 운영체제로부터 할당받은 자원을 프로세스끼리 공유할 수 없기때문
  • 한 쓰레드가 종료되면 함께 동작하던 모든 쓰레드가 종료된다. 쓰레드는 프로세스 내부동작 단위로, 할당받은 자원들 중 stack을 제외한 모든 자원을 공유한다.그래서 한 쓰레드가 작동중 에러가나면 다같이 종료 된다.
