Build your own React (Step 0,1,2)

AUSG 5기 스터디 FE-Deep-Dive 중 Build your own react를 참고해 작성된 포스트입니다.

Step 0

React, JSX, DOM 요소가 작동하는 법을 기존의 바닐라코드로 대체해보며 아주 작은코드로 간단하게 설명합니다.

const element = <h1 title="foo">Hello</h1>;
const container = document.getElementById("root");
ReactDOM.render(element, container);

위에 3줄짜리 리액트 앱에서 리액트만의 코드를 제거하고 바닐라JS로 대체 해봅니다.

  1. JSX : const element = <h1 title="foo">Hello</h1>

    const element = <h1 title="foo">Hello</h1>;
// jsx -> React.createElement
const element = React.createElement("h1", { title: "foo" }, "Hello");

    JSX는 React.createElement라는 DOM 노드를 생성하기 위한 정보를 템플리 언어처럼 사용할 수 있도록 돕습니다. 사용이 필수는 아니지만 UI 관련 작업에서 시각적으로 매우 도움이 됩니다.

    * JSX문법을 바벨을 통해 React.createElement로 해석됩니다. 바벨 컴파일러로 직접 확인해보기

    const element = React.createElement("h1", { title: "foo" }, "Hello");
// return React.createElement
const element = {
  type: "h1",
  props: {
    title: "foo",
    children: "Hello",
  },
};

    createElement는 Step1에서 조금 더 자세하게 알아봅니다.

  2. ReactDOM.render(element, container)

    const node = document.createElement(element.type)
node["title"] = element.props.title
​
const text = document.createTextNode("")
text["nodeValue"] = element.props.children

node.appendChild(text)
container.appendChild(node)

    ReactDOM.render에서 지정된 container(html파일의 id가 root인 태그)에 위에서 만든 element로부터 DOM 요소의 정보를 받아와 생성합니다.

다음 Step 1,2에서 Step 0에서 간단히 본 React.createElement와 RecatDOM.render를 직접 만들어 봅니다. 실제 React코드의 아키텍처를 따르지만 전체를 구현하는건 아닙니다. 간단한 코드로 단순화 하여 편집된 코드입니다.

const element = (
  <div id="foo">
    <a>bar</a>
    <b />
  </div>
);
const container = document.getElementById("root");
ReactDOM.render(element, container);

위 코드를 기준으로 만들어봅니다.

Step 1: The createElement Function

리액트의 createElement

React.createElement(type, config, children);

// React.createElement 반환 값
ReactElement(
  type,
  key,
  ref,
  props
  /*...*/
);
  • type : div, span, h1과 같은 DOM노드로 만들어질 html 태그를 정의하거나(React.Component), React.Fragment(<></>로 사용하기도 함)가 들어가기도 합니다.
  • config : 자식 요소에게 넘길 값(함수, 변수 등등)입니다. 객체로 존재하여 입력된 속성을 key와 value로 넘깁니다. 예약 속성으로 key와 ref가 존재합니다.
  • children : props로 반환되는 특별한 속성입니다. 배열로 여러가지가 들어가기도 하고, string으로 넘어가기도 합니다.

createElement를 통해 받은 인자값들은 ReactElement를 반환합니다. config로 입력받은 객체에서 ref와 key속성은 따로 전달되며, 이 둘을 제외한 모든 속성은 props객체로 들어갑니다.

추가로 따로 전달 받은 children은 arguments의 수에 따라 type과 config를 제외하고 1개일때는 props.children에 그대로 들아가며, 그 이상일때는 배열로 생성되어 들어갑니다.

createElement 직접 구현하기

위에서 본 react의 구조를 간단하게 구현합니다.

  • createTextElement : children배열에 element생성 시 객체가 아닌 primitive 값을 위한 특별한 유형(TEXT_ELEMENT)을 생성해 함수를 만듭니다.
// children에 인자로 string이 넘어왔을 때 wrapper함수
function createTextElement(text) {
  return {
    type: "TEXT_ELEMENT",
    props: {
      nodeValue: text,
      children: [],
    },
  };
}
  • createElement : 위에서 봤던대로 props는 그대로 전달하고, children은 배열로 생성하며 위에서 만든 primitive 값을 위한 구분정도를 해줍니다.
// 직접 만들어본 createElement
function createElement(type, props, ...children) {
  return {
    type,
    props: {
      ...props,
      children: children.map((child) =>
        typeof child === "object" ? child : createTextElement(child)
      ),
    },
  };
}
* 실제 React에서 primitive 값을 위해 래핑 하거나 children을 위해 빈 배열을 생성하진 않습니다. props에서 ref나 key와 같은 속성은 무시하고 진행합니다. 
// 우리가 구현한 기능들
const Didact = {
  createElement,
};

const element = Didact.createElement(
  "div",
  { id: "foo" },
  Didact.createElement("a", null, "bar"),
  Didact.createElement("b")
);

const container = document.getElementById("root");
ReactDOM.render(element, container);

추가로 우린 JSX를 사용하려 합니다. /** @jsx Didact.createElement */로 코멘트를 추가시 바벨이 JSX 변경할때 우리가 정의한 함수를 사용해줍니다.

/** @jsx Didact.createElement */
const element = (
  <div id="foo">
    <a>bar</a>
    <b />
  </div>
);

Step 2 : The render Function

이번엔 render를 구현합니다. 직접 DOM에 그려지는 단계입니다. DOM을 추가하는데 만 집중합니다. 다른 추가적인 기능은 고려하지 않습니다.

// createElement 생략
// ...

function render(element, container) {
  // dom을 생성하는 단계, step1에서 단순 createTextNode일때와 구분
  const dom = element.type == "TEXT_ELEMENT"
      ? document.createTextNode("")
      : document.createElement(element.type)
​
// props에 있는 값들을 dom의 속성으로 추가하기위한 단계,
// key값중 Children은 요소를 만들기 위함이였기 때문에 필터 후
// 남은 속성들을 dom의 속성으로 추가합니다.
const isProperty = key => key !== "children"
  Object.keys(element.props)
    .filter(isProperty)
    .forEach(name => {
      dom[name] = element.props[name]
    })

  // children에 있는 컴포넌트들도 재귀로 render
  ​element.props.children.forEach(child =>
    render(child, dom)
  )
  container.appendChild(dom)
}

const Didact = {
  createElement,
  render, // 추가 완료!
}

Didact.render(element, container)
