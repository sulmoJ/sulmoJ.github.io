상태관리 1

상태 관리

프론트엔드에서 엔터프라이즈급 어플리케이션을 만들기위해 상태관리 툴은 거의 필수적인 느낌입니다. 무지성으로 사용하던 Redux나 Vuex같은 상태관리 툴의 동작방식을 알아보는 좋은 기회가 될것 같습니다.

상태관리 툴을 사용하는 대표적인 이유로 상태관리가 복잡해지고 컴포넌트의 Depth가 깊어져 props로 단방향 처리나 이벤트 함수를 통해 올려보내는 식의 상태조작은 한계가 있습니다.

이 때 중앙 집중식 저장소 역할을 하면 예측 가능한 방식으로 상태를 변경 할 수 있도록 도와주는 상태관리 툴들의 기본적인 동작 방식을 알아봅니다.

Observer Pattern? 발행/구독 패턴(Pub/Sub)?

이 둘의 패턴이 같다라고 하는 글도 보였고 반대다?라는 글도 보여서 혼란스럽습니다. 뭔가 각잡고 파지 않으면 이상하게 기억될 것 같아 무서운데.. 일단 간단하게 이해한 느낌은

Observer Pattern

  • 결합도가 높은데 이유가 Subject에 Observer들이 등록되어 직접 알려줌으로 서로를 인지하고 있어야 하기 때문인것 같습니다.
  • 상태의 궁굼증과 상관없이 항상 통보받습니다.
  • 동기적으로 작동합니다.

Pub/Sub Pattern(발행/구독 모델)

  • Pub/Sub 패턴은 관찰자와 관찰대상 이 외에 추가적인 컴포넌트가 존재하는데 여기서 관찰자와 관찰대상의 소통을 담당한다.
  • 위와 같은 이유로 Observer Pattern보다 결합도가 낮다.

Object.defineProperty(object, prop, descriptor)

객체에 직접 새로운 속성을 정의하거나 이미 존재하는 속성을 수정한 후, 그 객체를 반환합니다.

위와 같은 정의를 가지고 있지만 우리가 여기에서 사용할 기능은 매개변수 descriptor에서 get, set을 이용해 값의 변화에 따라 관찰자를 통한 함수 실행입니다.

여러가지 속성이 있는데 이 부분은 MDN에서 확인해 봅시다!

(추가적으로 defineProperty는 IE를 지원하기위한 문법입니다. 최신 브라우저에서는 Proxy를 이용하면 더 쉽게 만들 수 있습니다. )

예제

const state = {
  a: 10,
  b: 20,
};

const stateKeys = Object.keys(state);
const observer = () => console.log(`a + b = ${state.a + state.b}`);

for (const key of stateKeys) {
  let _value = state[key];
  Object.defineProperty(state, key, {
    get() {
      return _value;
    },
    set(value) {
      _value = value;
      observer();
    },
  });
}

observer();

state.a = 100;
state.b = 200;

위의 코드를 통해 우리가 관찰하고 싶은 상태가 값이 변할 때마다 observer가 실행됨을 볼 수 있습니다.

이런 구조의 코드를 함수화 하여 중앙 저장소의 값을 관찰하면, 구독되어있는 컴포넌트들에 전역으로 쉽게 상태를 사용할 수 있습니다.

Flux 패턴

Vuex와 Redux에서 사용되는 패턴입니다. 특징은 기존 다른 패턴들과 다르게 단방향흐름을 가지고있습니다.

이전의 패턴들에서 문제는 대표적으로 Facebook 알림(notification) 버그가 있었다고 합니다. 양방향에 비동기적으로 변경이 생길 수 있었고 예측이 힘들어 디버깅이 어려웠던 근본적인 문제를 Flux를 통해 해결했다고 합니다.

여기에서 flux패턴설명을 재밌게 봤습니다.
