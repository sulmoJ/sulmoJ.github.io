OSI 7계층 정.리.


OSI 7계층

네트워크 통신 과정을 7개의 계층으로 구분한 산업 표준 참조 모델

네트워크 통신에 있어 OSI 7계층의 목적은 표준과 학습도구입니다.

여러 통신업체들 장비들이 자신의 업체 장비끼리만 연결이 되었고 호환성이 떨어졌습니다. 그래서 전세계 표준 제정 위원회에서 선정된 위원들로 구성된 국제 표준 기구에서 OSI 7계층을 정의했고 표준이 됩니다.

응용계층

가장 사용자랑 가장 가까운 계층으로 응용SW를 돕는 계층

가장 간단한 예로 한 서비스 서버에서 외부의 요청에 의해 html파일(Data)을 보내줘야 할때, 서버에서 전송을 위해 http에 맞춰 데이터를 캡슐화 합니다.

  • 프로토콜 : DHCP, DNS, FTP, HTTP

표현계층

송신층과 수신측 사이에 데이터형식을 정해준다. 받은 데이터를 코드변환, 구문 겁색, 암호화, 압축의 과정등을 올바른 표준 방식으로 변환해 줌

  • 프로토콜 : JPG, MPEG, SMB, AFP

세션계층

통신 세션을 구성하는 계층, 포트번호를 기반으로 연결한다.

동시송수신(Duplex), 반이중(Half-Duplex), 전이중(Full-Duplex) 방식의 통신과 함께 체크 포인팅과 유후, 종료, 다시 시작 과정 등을 수행합니다.

  • 프로토콜 : NetBIOS, SSH, TLS

전송계층

전체 메시지를 발신지 <-> 목적지 간 제어와 에러를 관리한다.

패킷들의 전송이 유효한지 확인하고 실패한 패킷은 다시보내는 등 신뢰성 있는 통신을 보장하며, 헤더에 세그먼트 가 포함됩니다.

  • PDU : 세그먼트
  • 프로토콜 : TCP, UDP
  • 장비 : 게이트웨이, L4 스위치

네트워크계층

기기에서 데이터그램(Datagram)이 가는 경로를 설정해주는 역할을 한다. 라우팅 알고리즘을 사용하여 최적의 경로를 선택하고 송신측으로부터 수신측으로 전송한다.

  • PDU : 패킷(Packet)
  • 프로토콜 : IP, ICMP 등
  • 장비 : 라우터, L3 스위치

링크계층

네트워크 기기들 사이의 데이터 전송을 하는 역할

스위치 같은경우 MAC주소를 통해 정확한 장치로 정보전달이 가능합니다. 3계층에서 정보를 받아 주소와 제어정보를 헤더와 테일에 추가합니다.(다른 계층은 헤더만 있었음)

  • PDU : 프레임(Frame)
  • 프로토콜 : 이더넷, MAC, PPP, ATM, LAN, Wifi
  • 장비 : 브릿지, 스위치

물리계층

상위 계층에서 전송된 데이터를 물리 매체(허브, 라우터, 케이블 등)를 통해 다른 시스템에 전기적 신호를 전송하는 역할

진짜 데이터들을 0과1의 비트로, 기계어를 전기적 신호로 바꿔서 와이어에 실어줍니다.

  • PDU : 비트(Bit)
  • 프로토콜 : Modem, Cable, Fiber, RS-232C
  • 장비 : 허브, 리피터

캡슐화

이렇게 정리한 순서대로가 캡슐화가 되는 순서이고, 반대로 다시 많은 네트워크를 거쳐 응용프로그램이 원하는 데이터로 풀어가는게 역캡슐화라고 합니다.

TCP/IP 모델

위에서 정말 개론적으로 공부한 OSI는 말그대로 참조 모델이며 학습도구에 해당합니다.

현재 우리가 사용하는 인터넷 프로토콜 스택은 대부분 TCP/IP를 따릅니다.

이 모델은 데이터의 흐름 관리, 정확성 확인, 패킷의 목적지 보장을 담당합니다. 데이터의 정확성 확인은 TCP가, 패킷을 목적지까지 전송하는 일은 IP가 담당합니다.

OSI 7계층과 다르게 4계층으로 이뤄져있지만 응용계층에 표현,세션계층의 기능이 추가적으로 있는 모양입니다.

Data → Segment → Datagram → Frame DPU의 변화도 차이가 없습니다. 이러한 순서로 캡슐화를 진행하고 사용중에 있습니다.
