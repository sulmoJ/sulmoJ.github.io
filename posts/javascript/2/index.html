자바스크립트 실행 컨텍스트(Execution Context)

실행 컨텍스트

this를 공부하려다 실행컨텍스트까지 알아보게 되었습니다.. javascript엔진 작동과 여러가지가 한번에 들어가야하는 내용 이었지만 알게된김에 간단히만 정리하려고 노력 해봤습니다.

  • poiem web에서

    실행 컨텍스트(Execution Context)는 scope, hoisting, this, function, closure 등의 동작원리를 담고 있는 자바스크립트의 핵심원리이다.

  • ECMAScript 스펙에 따르면

    실행 가능한 코드를 형상화하고 구분하는 추상적인 개념 -

추상적 개념이라 글로만은 이해가 되지 않네요.

일단 위에 정의한 내용과 설명들을 뜯어보면,

  1. 실행 가능한 코드엔 전역 코드, 함수 코드, Eval 코드들이 있고,
  2. 해당 코드들이 scope, hoisting, this, function, closure과 같은 여러가지 작동 원리들에 의해 언제 어디서 작동할지를 도울 수 있도록 정보를 가지고 활용하는 것이

실행 컨텍스트로 이해를 했습니다.(아래 정보들을 공부하고)

코드로 예를 들면

var x = "xxx";

function foo() {
  var y = "yyy";

  function bar() {
    var z = "zzz";
    console.log(x + y + z);
  }
  bar();
}
foo();

를 실행시 아래처럼 실행 컨텍스트들의 스택이 생기고, 제어권이 이동하며 실행 컨텍스트들의 정보에서 참조하여 구동됩니다.

실행 컨텍스트가 물리적으로 가진 3가지 속성

실행 컨텍스트가 위 설명처럼 동작하기위해 여러가지 정보를 가지는데 거기서 대표적을 3가지 속성이 있습니다.

  • Variable Object
  • Scope Chain
  • this
Variable Object(변수 객체 / VO)

실행 컨텍스트가 생성되면 js엔진은 실행에 필요한 여러가지 정보를 담을 객체를 생성합니다. 이게 VO이고, 엔진에 의해 참조되고 코드에서는 접근할 수 없습니다.

VO가 가지는 정보들

  • 변수
  • 매개변수와 인수 정보
  • 함수 선언

이게 또 컨텍스트의 종류에따라 가르키는 변수객체의 종류가 다르다고 합니다.

  • 전역 컨텍스트의 경우 => 전역 객체(전역에 선언된 전역 변수와 전역 함수를 프로퍼티로 소유)
  • 함수 컨택스트의 경우 => 활성 객체(매개변수와 인수들의 정보를 배열의 형태로 담고 있는 객체인 arguments object가 추가)

스코프 체인

일종의 리스트로 자신과 연관되어있는 스코프들을 차례로 저장해 놓습니다.

스코프의 연결 같은 느낌인데, 위의 foobar 예시코드를 예로들면 bar 함수가 x라는 변수를 참조할때 스코프 체인이라는 리스트의 정보를 활용합니다.

  1. 일단 처음에 자신의 함수 스코프에서 x를 찾고 (결과: 없음)
  2. 다음으로 스코프체인 리스트에 있는 다음 범위의 스코프에서 x를 찾습니다.(foo의 스코프에 해당하며, 결과: 없음)
  3. foo에도 없다면 가장 바깥인 글로벌에 스코프에 해당하며 변수를 참조합니다.(결과 : x 찾음)

스코프 체인에서 리스트의 값으로 범위에 컨텍스트의 VO(Variable Object)를 가르킨다고 합니다.
