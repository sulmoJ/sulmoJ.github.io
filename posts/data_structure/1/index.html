Tree의 시작


Tree

특징

  • 비선형 자료구조
  • 계층적 관계

표현에 집중된 자료구조라 한다. 무엇인가 저장하고 꺼낸다는 사고에서 벗어나 트리라는 자료구조를 볼 필요가 있다.

용어

  • Node (노드) : 트리를 구성하고 있는 각각의 요소를 의미한다.
  • Edge (간선) : 트리를 구성하기 위해 노드와 노드를 연결하는 선을 의미한다.
  • Root Node (루트 노드) : 트리 구조에서 최상위에 있는 노드를 의미한다.
  • Terminal Node ( = leaf Node, 단말 노드) : 하위에 다른 노드가 연결되어 있지 않은 노드를 의미한다.
  • Internal Node (내부노드, 비단말 노드) : 단말 노드를 제외한 모든 노드로 루트 노드를 포함한다.

이진트리(Binary Tree)

모든 노드가 정확히 두개의 서브트리를 가지는 트리

  • 추가로 서브트리의 왼쪽과 오른쪽은 확실하게 구분되며
  • 서브트리는 공백이 될 수 있다.

  • Binary Tree는 노드의 개수가 n 개이고 root가 0이 아닌 1에서 시작할 때 i 번째 노드에 대해서 아래의 인덱스를 가진다.

    • parent(i) = i/2
    • left_child(i) = 2i
    • right_child(i) = 2i + 1

포화 이진트리(full binary tree)

이진트리가 보유할 수 있는 최대의 노드를 가진 형태

높이 별 존재할수 있는 노드는 다 존재하는 상태이다. 그래서 높이별 최대 노드수를 알수 있다. 2^(h+1) - 1

완전 이진트리(complete binary tree)

마지막 층을 제외한 형태에서 포화 이진트리이며, 마지막 층을 왼쪽부터 채운 트리


이진 탐색 트리(Binary Search Tree)

데이터를 저장하는 규칙이 존재하는 이진트리

규칙
  1. 이진 탐색 트리의 노드에 저장된 키는 유일하다.
  2. 부모의 키가 왼쪽 자식 노드의 키보다 크다.
  3. 부모의 키가 오른쪽 자식 노드의 키보다 작다.
  4. 왼쪽과 오른쪽 서브트리도 이진 탐색 트리이다.

키값에 따라 특정 값을 찾는데 도움이 되지만 저장 순서에 따라 편향 이진 트리가 될수 있다. 이 문제점을 해결 하기위 해 AVL Tree, Red-Black Tree가 존재한다.

위의 문제점에의해 시간복잡도 의 차이가 있는데

  • 균등 트리 : O(log N)
  • 편향 트리 : O(N) * 노드 개수가 N개일 때
